UltraVanilla Non-Vanilla Features Vote

We are considering adding some of these features that have been suggested numerous times over the past 3 years. This is a survey to vote on these matters.

Below we show comments collected on our previous survey. We show these comments as a way to elevate concerns & strong opinions. Think of it like a voter information booklet or a comments-on-proceedings sort of thing. If you want any comments added last minute, let us know.

Consider also joining the discussion in #suggestions too! Run /discord ingame and click the link to access this chat.

Voting rules update 2022-05-25: We didn't expect to do a reddit ad for the server during voting, but we do still want to count the new player vote. So as a compromise, votes will only be counted if the user who submitted has 1h30m hours of play time by the end of the week. Please consider only voting if you intend to stick around the server. All of the votes before this rules change meet this requirement, so if you already voted then this doesn't affect you.

Proposal #1: Other mob heads

This adds custom mob heads for all of the passive mobs, as well as hostile mobs that don't have an 8x8 head. When another mob is killed by a charged creeper explosion, their head will drop. The heads use the "MHF_" textures that a Mojang employee made a few years ago.

Proposal #2: Terracotta screwdriver

When right clicking a block with a brick, it will rotate it. Its functionality will be hardened to avoid creating invalid block states, and will be able to record block changes to coreprotect to avoid griefing.

This way, you won't have to constantly break and re-place glazed terracotta to get the right positioning.

Note: The below gif is actually incorrect, it won't be able to create impossible corner stairs, rather it will flip to the appropriate shape state according to the blocks around it.

Proposal #3: Husks hold sand

This would make husk zombies occasionally hold sand, so that a mob farm built in a desert can automate the production of sand.

Because this may impact the economy, we will put a supermajority vote threshold of 60% for this suggestion to be implemented.

Proposal #4: Phantoms are tied to the user that spawned them (PhantomSMP)

Phantoms will ignore players who used a bed in the last 3 Minecraft days, even in crowded areas. Also, phantoms will instantly despawn when all players in an area have slept in a bed, rather than lingering until day.
